### v1.2 (예상 1개월) 📅 계획
**고급 날짜 처리**

#### 기능 명세:
메모에서 날짜 텍스트를 감지하여 자동으로 Notion의 날짜 필드 업데이트

**지원 형식:**
```
❌ 현재: 2026-04-13 (생성 시간만)

✅ v1.2에서 지원:
- ISO 형식: "2026-04-13", "2026-04-13T15:30:00"
- 한국식: "4월 13일", "2026년 4월 13일"
- 상대 날짜: "내일", "3일 뒤", "다음 주 월요일"
- 범위: "2026-04-13 ~ 2026-04-20"
```

**사용 예시:**
```
📌 프로젝트 데드라인
  #업무관련 #프로젝트

  메모:
  기한: 2026-04-20
  →Notion의 날짜 필드: 2026-04-20으로 설정
```

**구현:
```
src/date_parser.py (신규)
└─ parse_date_text(text, base_date)
   ├─ ISO 형식 파싱
   ├─ 한국식 날짜 파싱 (pytz, dateutil)
   ├─ 상대 날짜 해석 (word2number)
   └─ 신뢰도 점수 반환 (confidence_score)
```

**테스트:**
```bash
pytest tests/test_date_parser.py -v
```

### v2.0 (예상 3개월) 🚀 주요 업그레이드
**Dynalist 계층 구조 개선**

#### 기능 명세:
첫 번째 태그를 분류 기준으로 사용하고, 그 아래 child 노드들을 계층별로 매핑

**현재 vs 예정:**

**현재 (v1.0):**
```
📌 자기계발 계획
  #개인용무 #자기계발 #독서 #코딩
  
메모: ...
```

→ 평면 처리
- 대분류: #개인용무
- 중분류: #자기계발
- 소분류: [#독서, #코딩]

---

**v2.0에서:**
```
📌 자기계발 계획
  #개인용무 (첫 태그 = 분류 기준)
  ├─ 자기계발 노드
  │  ├─ 독서 노드
  │  └─ 코딩 노드
  └─ 건강 노드
     └─ 운동 노드
```

→ 계층 구조 반영
- DB 분류: #개인용무 → personal DB
- 대분류: #자기계발 (첫 번째 child의 태그)
- 중분류: #독서 (두 번째 child의 태그)
- 소분류+: [나머지 tags...]

**구현 변경사항:**
```python
# transformer.py 수정
class Transformer:
    def _build_hierarchical_tags_v2(self, 
                                   item_node, 
                                   node_map, 
                                   parent_map):
        """
        v2.0: Dynalist의 실제 child 계층을 반영
        
        1. 첫 태그 추출 → DB 분류 기준
        2. child 노드 탐색 → 계층 구조 맵핑
        3. 무한 depth 지원 → 4단계 이상은 소분류에 누적
        """
```
